# TaskFlow Single

Используй этот skill, когда хочешь **одной командой** провести задачу от постановки до поэтапной реализации, сохраняя минимальный набор артефактов и высокое качество.

## База
- Работай в каталоге задачи `.tasks/<slug>/` (slug можно `NN-alias` или `YYYYMMDD-HHMM--alias`).
- Любой текстовый документ — **максимум 1 страница**. Всё, что не влезло: либо упростить, либо вынести в отдельный appendix/таблицу.
- Для рефакторингов/миграций делай работу **шагами (сабтасками)**: каждый шаг имеет deliverable + check + rollback.
- **Anti-slop** применяй **после каждого шага**, а не в конце.
- Для discovery сначала используй локальный корпус `~/.agents/references` (релевантные подпапки по стеку задачи), затем внешние источники.

## Флоу (кратко)
1) Создай структуру задачи и заполни исходную постановку.
2) Сними текущее состояние системы (AS-IS) + инвентарь (модели/эндпоинты/точки авторизации/FE-вызовы).
3) Проведи discovery research: найди **1–3 кандидата**, покажи fit, trade-offs и сравнительную таблицу. **Не выбирай кандидата сам — остановись и спроси пользователя.** Только после явного выбора двигайся дальше.
4) Зафиксируй "как должно работать" через success checks + инварианты.
5) Подготовь артефакты для изменения: модель/связи, миграции, data flow (если применимо).
6) Составь план сабтасков реализации. Сабтаск = шаг разработки (код, миграция, флаг, деплой). Дискавери, проектирование и составление плана — **не сабтаски**. Не создавай сабтаски до этого шага.
7) Выполняй сабтаски по одному, проверяя и фиксируя результат каждого.

Подробности и шаблоны — в `references/`.
