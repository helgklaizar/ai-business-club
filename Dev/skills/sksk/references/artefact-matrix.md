# Матрица артефактов (минимальная)

Правило: если артефакт не применим — пиши `N/A` и короткую причину в `2-plan.md`.

Легенда:
- **обязательно** — без этого качество падает резко.
- **по ситуации** — включай при условиях.

| Артефакт | Когда делать | Рекомендация |
|---|---|---|
| Постановка задачи | Всегда | обязательно |
| План-индекс (ссылки на артефакты + сабтаски) | Всегда, кроме микро-фикса | обязательно |
| AS-IS one-pager | Любая задача, где меняется логика/данные/контракты/права | обязательно |
| Инвентарь (модели, маршруты, FE-вызовы) | Если есть код и нужно “вписаться в систему” | обязательно для рефакторингов/прав |
| Discovery research: 1–3 кандидата + fit | Любая задача, где ты не уверен в лучшем подходе или система нетривиальна | обязательно для архитектурных/рефакторинговых |
| Success checks + инварианты | Любая задача, где меняется поведение | обязательно |
| MODEL (сущности/связи) | Меняются модели/таблицы/связи | обязательно |
| MIGRATIONS (порядок/rollback) | Есть миграции/перенос данных/dual-write | обязательно |
| DATAFLOW (клиент→API→auth→DB) | Меняется контекст/авторизация/сложные запросы/несколько акторов | по ситуации (часто нужно) |
| Сабтаски | Рефакторинг/миграции/поэтапная выкладка | обязательно |
| ADR (короткое решение + почему) | Долгоживущее/спорное решение | по ситуации |
| Итог (result) | Всегда | обязательно |
