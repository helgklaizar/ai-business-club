Это действие выполняется АВТОМАТИЧЕСКИ после завершения каждой задачи! Без дополнительных вопросов.

## Шаги (Post-task):

// turbo-all
1. Проанализировать изменения в коде (измененные файлы, закрытые/добавленные задачи).
2. Запустить все доступные тесты в проекте (убедиться, что сборка не сломана). Если тесты не проходят, исправить ошибки.
3. Обновить документацию ВЕЗДЕ, где нужно (в коде, README.md, `project-info.md` и др.).
4. Сделать `git commit` с понятным сообщением (формат `тип: описание`) и отправить изменения в удаленный репозиторий (`git push`).
5. Запустить актуальную сборку проекта (сделать новый билд).
