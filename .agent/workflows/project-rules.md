# Память проекта
- **Статика (`project-info.md`)**: архитектурные решения, стек, пути для запуска. Дополнять при глобальных изменениях.
- **Динамика (`GEMINI.md`)**: оперативная память проекта. ОБЯЗАТЕЛЬНО обновлять в конце каждой задачи или при сменах фокуса (вписывать что сделано и что делать дальше).
- Цель: следующая сессия стартует за 1 секунду без потери контекста.

# Git правила
- Формат:  -> , 
- Типы коммитов:      
- Один коммит = одна логическая правка.
- Делайте коммит/пуш автоматически как часть рутины закрытия задачи.

# Система апдейтов (для Desktop/Mobile Apps)
- В каждом проекте с приложением должна быть кнопка в шапке (зеленая) "Update" (или подобный UI).
- При наличии изменений в локальном коде (своими силами или ИИ), кнопка "Update" инициирует автоматический апдейт скриптом (пересборка) и перезапуск приложения. Не требовать ручного удаления и копирования билда каждый раз.
- Строгий контроль версий: после каждого апдейта ИИ инкрементирует только минорную цифру в конце. Версия обязана сохранять формат SemVer (например, ). Основные цифры (1.0.0) и статус (beta) назначает только пользователь.
# Запрещено / Ограничения
- Удалять файлы без явного подтверждения.
- Выполнять деструктивные операции (drop, очистка, перезапись) без спроса.
- Пушить в main/master без подтверждения.
- Устанавливать глобальные зависимости без спроса.
- Перемещать, переименовывать или реструктурировать код который не просили трогать.
- Добавлять улучшения, рефакторинг или новшества сверх того что явно попросили.
