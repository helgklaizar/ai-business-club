# AI Business Club

## Общая информация
Закрытый Phygital (онлайн+офлайн) клуб для владельцев бизнеса и сотрудников, сфокусированный на практической интеграции ИИ в бизнес-процессы.
**Основная специфика:**
- Доступ только по ежемесячной подписке (Провайдер оплаты будет выбран позже).
- Киллер-фича: Библиотека реальных кейсов внедрения с ROI (сэкономленные деньги/время).

## Стек технологий
- **Фреймворк:** Next.js (App Router, React)
- **Стилизация:** Tailwind CSS
- **База данных & Бэкенд:** Firebase (Firestore)
- **Аутентификация:** Firebase Auth (Email, Google)
- **Биллинг (Подписка):** К выбору (израильский провайдер)

## Локальный запуск
1. Установите зависимости: `npm install`
2. Запустите dev-сервер: `npm run dev`
3. Убедитесь, что настроены переменные окружения в `.env.local` (Firebase).

## Структура задач (SKSK Методология)
Проект поделен на последовательные задачи в директории `.tasks/`:
- **01-setup-firebase**: Подключение Firebase (Auth, Firestore).
- **02-landing-and-auth**: Верстка лендинга (заглушка) и форма логина/регистрации (Firebase Auth).
- **03-cases-and-paywall**: Дашборд "Success Stories" и логика Пейвола (нет подписки в БД — показать окно оплаты).
- **04-vault-prompts**: Каталог промптов и обучающих материалов для резидентов.
- **05-academy-events**: Календарь событий (воркшопы) и архив записей.
- **06-community-chats**: Интеграция с комьюнити (чат/дискорд) с проверкой подписки.
- **07-admin-panel**: Панель администратора для управления контентом и доступом.

## Важные архитектурные решения
- Все изменения выполняются строго в рамках методологии SKSK.
- Монетизация: жесткий пейвол. Нет активной подписки = нет доступа к контенту базы знаний и чатам.
- Firebase Firestore используется для хранения данных профиля, статуса подписки и контента базы знаний.
